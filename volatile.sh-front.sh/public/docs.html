Technical guide for volatile.sh

Quick Start

  1. Go to volatile.sh
  2. Type or paste your secret into the text area
  3. Select an expiration time (5 minutes to 7 days)
  4. Click "GENERATE LINK"
  5. Copy the link and share it securely
  6. The recipient opens the link to read the secret once
  7. The secret is permanently deleted after reading

How It Works

volatile.sh uses a zero-knowledge architecture:

  1. Key Generation: Your browser generates a random AES-256-GCM key
  2. Client Encryption: Your secret is encrypted locally before any data leaves your device
  3. Key in Fragment: The decryption key is embedded in the URL fragment (#key), which browsers never send to servers
  4. Server Storage: Only the encrypted ciphertext and IV are stored on the server
  5. One-Time Read: When retrieved, the secret is deleted immediately

API Reference

Health Check

GET/api/health
Response: { "ok": true }

Create Secret

POST/api/secrets 
Request Body:
{
  "encrypted": "base64url-encoded ciphertext",
  "iv": "base64url-encoded IV (12 bytes)",
  "ttl": 86400000  // Optional: milliseconds, defaults to 24 hours
}

Response:
{
  "id": "aB3xY9",
  "expiresAt": 1704067200000
}

Read Secret

GET/api/secrets/:id 
Response (first read):
{
  "encrypted": "base64url-encoded ciphertext",
  "iv": "base64url-encoded IV"
}

Response (already read or expired):
{ "error": "Secret not found" }

Client-Side Encryption Example

// Generate a key
const key = await crypto.subtle.generateKey(
  { name: "AES-GCM", length: 256 },
  true,
  ["encrypt", "decrypt"]
);

// Encrypt your message
const iv = crypto.getRandomValues(new Uint8Array(12));
const encoded = new TextEncoder().encode("Your secret here");
const ciphertext = await crypto.subtle.encrypt(
  { name: "AES-GCM", iv },
  key,
  encoded
);

// Send to server (key never leaves your browser!)

Rate Limits

To prevent abuse, the following rate limits apply per IP address:

Action Limit Window
Create secret 100 1 hour
Read secret 1,000 1 hour

Error Codes

Code Status Description
MISSING_FIELDS 400 Required fields not provided
INVALID_ENCODING 400 Data must be base64url encoded
INVALID_IV_LENGTH 400 IV must be 12 bytes
SECRET_TOO_LARGE 413 Secret exceeds ~1MB limit
RATE_LIMITED 429 Too many requests
NOT_FOUND 404 Secret doesn't exist or was burned
INTERNAL_ERROR 500 Server error

Browser Requirements

volatile.sh requires a browser that supports:

  • Web Crypto API - For client-side encryption
  • Secure Context (HTTPS) - Required for Web Crypto
  • Modern JavaScript - ES2020+ features

Supported browsers: Chrome 90+, Firefox 88+, Safari 15+, Edge 90+

Self-Hosting

volatile.sh is built on Cloudflare Workers and Durable Objects:

  1. Fork the repository on GitHub
  2. Install dependencies: npm install
  3. Configure wrangler.toml with your Cloudflare account
  4. Build the frontend: cd volatile.sh-front.sh && npm run build
  5. Deploy: npx wrangler deploy

Security Best Practices

  • Always share the full link including the fragment (everything after #)
  • Share links through secure channels (encrypted messaging, in person)
  • Avoid using link preview features in chat apps
  • Be aware that anyone with the link can decrypt the secret
  • Consider the recipient's environment—do they have screen recording or monitoring?
